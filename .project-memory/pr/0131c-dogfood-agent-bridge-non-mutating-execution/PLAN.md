# PR 0131C — Dogfood Agent Bridge Non-Mutating Execution Fix

## Root Cause

The dogfood pipeline fails with `reason_codes=["git_mutation_not_allowed"]` at
planner, plan-review, and coder bridge steps (diagnostic from the current
code state, not from a test run of this branch).

### Exact Blocker Source

**File:** `services/runner/src/runner/agent_runner_bridge.py`

**Function:** `_check_forbidden_patterns()` (lines 201–222), called at ~line 247.

**Called from:** `run_agent_runner_bridge()` with the full `task_prompt`.

**The chain:**

1. `pipeline_runner.py` calls `run_agent_runner_bridge(agent_name, task_prompt, ...)`,
   where `task_prompt` is the full composed prompt text from `PromptPacket.prompt_text`.
2. The prompt text is generated by `prompt_composer.py` and includes a safety
   instructions block produced by `_forbidden_commands_block()` that contains
   exact lines like:
   ```
   Forbidden commands:
   - git add
   - git commit
   - git push
   ...
   ```
3. `run_agent_runner_bridge()` calls `_check_forbidden_patterns(task_prompt, codes)`.
4. `_check_forbidden_patterns` checks:
   - `_FORBIDDEN_HIDDEN_REASONING_PATTERNS` (imported from `improvement_backlog.py`)
   - **`_FORBIDDEN_ACTION_PATTERNS` (imported from `improvement_backlog.py`)** — includes
     `("git commit", REASON_GIT_MUTATION_NOT_ALLOWED)`,
     `("git push", REASON_GIT_MUTATION_NOT_ALLOWED)`,
     `("git merge", REASON_GIT_MUTATION_NOT_ALLOWED)`,
     `("git rebase", REASON_GIT_MUTATION_NOT_ALLOWED)`,
     plus `pip install`, `npm install`, `import openai`, `subprocess.run`, etc.
   - `_FORBIDDEN_MUTATION_PATTERNS` (local to `agent_runner_bridge.py`) — includes
     `git add`, `git commit`, `git push`, `gh pr create`, etc.

**The match:** The `_FORBIDDEN_ACTION_PATTERNS` scan finds `git commit` / `git push`
in the safety-instruction block of the prompt text and returns
**`REASON_GIT_MUTATION_NOT_ALLOWED`** — the exact reason code from the dogfood
diagnostic.

Even if `_FORBIDDEN_ACTION_PATTERNS` were removed, `_FORBIDDEN_MUTATION_PATTERNS`
would still match the same strings (with `REASON_HIDDEN_REASONING_NOT_ALLOWED`
instead, which is the wrong code name but would still block).

### Why This Is a False Positive

The scanner is matching against **descriptive safety text** in the composed
prompt — not against actual executable command plans or agent config directives.
The prompt text is generated by the trusted Prompt Composer to instruct the agent
what NOT to do. The agent never receives real git/gh tools in its environment.

## Fix: Option B — Remove Over-Broad Scanner

**Chosen option:** Remove the `_FORBIDDEN_ACTION_PATTERNS` import/check and
the local `_FORBIDDEN_MUTATION_PATTERNS` tuple/check from
`agent_runner_bridge.py`. Keep the `_FORBIDDEN_HIDDEN_REASONING_PATTERNS` check
only (for `<cot>`, `<chain_of_thought>` — patterns that never appear in
generated prompt safety instructions).

**Reasoning:**
- The scanner on `task_prompt` was designed for backlog item text validation
  (`improvement_backlog.py`), not for composed agent prompts.
- The actual mutation safety is structural:
  1. The agent environment (Docker/local harness) does not provide real git/gh
     tools.
  2. Git Boundary is the only component authorized for git add/commit/push.
  3. `PromptPacket.forbidden_commands` metadata carries the restriction for any
     future adapter-level enforcement.
- The `_FORBIDDEN_MUTATION_PATTERNS` in `agent_runner_bridge.py` is duplicative
  and would also produce false positives against the same prompt safety text.
- Hidden-reasoning patterns (`<cot>`) are not present in generated prompt text
  and the check is harmless to retain.

**Update `_check_forbidden_patterns`** (lines ~210-222):

From:
```python
def _check_forbidden_patterns(text: str, codes: list[str]) -> None:
    """Check for forbidden patterns in text."""
    # Hidden reasoning
    for pattern in _FORBIDDEN_HIDDEN_REASONING_PATTERNS:
        if pattern in text:
            codes.append(REASON_HIDDEN_REASONING_NOT_ALLOWED)
            return

    # Forbidden actions (command execution, provider call, git mutation)
    for pattern, reason in _FORBIDDEN_ACTION_PATTERNS:
        if pattern in text:
            codes.append(reason)
            return

    # Forbidden mutation patterns (git commands, gh commands)
    for pattern, reason in _FORBIDDEN_MUTATION_PATTERNS:
        if pattern in text:
            codes.append(reason)
            return
```

To:
```python
def _check_forbidden_patterns(text: str, codes: list[str]) -> None:
    """Check for forbidden patterns in text.

    Only hidden-reasoning patterns are checked here.  Mutation and action
    patterns are not scanned against the task prompt because the prompt
    text is composed by the trusted Prompt Composer and includes safety
    instructions that describe forbidden commands as *descriptive text*,
    not as executable directives.  Structural git-mutation safety is
    enforced by:
    - The agent environment (no real git/gh tools)
    - Git Boundary (sole component authorized for git add/commit/push)
    - ``PromptPacket.forbidden_commands`` metadata
    """
    for pattern in _FORBIDDEN_HIDDEN_REASONING_PATTERNS:
        if pattern in text:
            codes.append(REASON_HIDDEN_REASONING_NOT_ALLOWED)
            return
```

class TestNonMutatingPromptAllowed:
    """Prompt text containing git mutation references is allowed."""

    def test_git_mutation_in_prompt_not_blocked(self, tmp_path: Path):
        """Git command text in prompt no longer triggers git_mutation_not_allowed."""
        agents_dir = _agents_dir(tmp_path)
        prompt = (
            "Implement a feature.\n"
            "Forbidden commands:\n"
            "- git commit\n"
            "- git push\n"
            "- pip install\n"
            "- subprocess.run\n"
        )
        result = run_agent_runner_bridge(
            agent_name="test-agent",
            task_prompt=prompt,
            agents_dir=agents_dir,
            allow_docker=False,
            output_dir=str(tmp_path),
            clock_provider=_clock_provider,
        )
        # Should not be FAILED — should proceed to Docker-blocked (BLOCKED)
        assert result.status == AgentRunnerBridgeStatus.BLOCKED
        assert REASON_DOCKER_BLOCKED in result.reason_codes
        # Should NOT contain git_mutation_not_allowed or hidden_reasoning_not_allowed
        for rc in result.reason_codes:
            assert "git_mutation" not in rc
            assert "hidden_reasoning" not in rc
            assert "action" not in rc

## Non-Mutating Artifact-Write Contract

- Agents may write artifacts to `PromptPacket.expected_output_path` (a
  deterministic allowlisted path under `.project-memory/pr/<pr-id>/`).
- Agents must not execute git/gh/Docker/pip commands.
- `agent_runner_bridge.py` no longer scans the composed prompt text for
  forbidden patterns.  The prompt text is trusted (generated by Prompt
  Composer).  Descriptive safety instructions in the prompt do not
  constitute executable command plans.
- Structural git-mutation safety is enforced by:
  1. The agent execution environment (no real git/gh tools available).
  2. Git Boundary — the sole component authorized for git add/commit/push.
  3. Human approval required for Git Boundary execution.

## Git Mutation Boundary

- **Not changed:** `services/runner/src/runner/git_boundary.py` — no

- **Not changed:** `services/runner/src/runner/run_persistence.py` — no

## Dogfood Validation

The dogfood pipeline must now be able to:
1. `compose_prompts` — completed (no change needed).
2. `planner` bridge — should no longer fail with `git_mutation_not_allowed`.
3. `planner_artifact_check` — requires existing PLAN.md (may pass if
   already present, or fail — but the bridge no longer blocks).
4. `plan_review` bridge — should no longer fail with `git_mutation_not_allowed`.
5. `plan_review_gate` — requires existing `plan-review.yml`.
6. `coder` bridge — should no longer fail with `git_mutation_not_allowed`.
7. `precommit_review` bridge — should no longer fail.
8. Pipeline stop now occurs only when artifacts/gates genuinely require
   human input, not due to a scanner false positive on safety text.

After this fix, dogfood will stop at the first genuine blocker (e.g.,
missing planner artifact, gate stop, or coder failure) rather than blocking
all agents before they can produce any output.
