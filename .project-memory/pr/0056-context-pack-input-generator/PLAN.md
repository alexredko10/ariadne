# PR 0056 — Context Pack Input Generator Plan

## Goal

Plan a minimal deterministic generator for context-pack-inputs artifacts.

The generator should convert explicit caller-provided task/workspace inputs into a canonical context-pack-inputs object.

This is the first small executable layer after Context Steward contracts and workspace artifact templates.

## Architectural Thesis

Ariadne should generate context inputs before compiling context.

Context-pack inputs are the boundary between Context Steward memory and Context Compiler behavior.

The model is replaceable.
The context input contract is durable.

This PR adds generation of structured inputs only.
It does not compile the final context pack.

## Existing Contract Snapshot

### Context Pack Inputs Schema (`schemas/context-pack-inputs.schema.yml`)

Defines 20 fields: pr_id, feature_id (optional), task_goal, source_contracts, relevant_anchors, allowed_paths, forbidden_paths, cache_key_refs, prior_pr_refs, qa_evidence_refs, known_risks, manual_checks_required, context_freshness (status + last_verified_hook), invalidation_inputs, requested_context_sections, output_preferences, created_from, updated_by.

## Generator Contract

### Pure function API

```python
def build_context_pack_inputs(
    *,
    pr_id: str,
    feature_id: str | None = None,
    task_goal: str,
    source_contracts: list[str] | None = None,
    relevant_anchors: list[str] | None = None,
    allowed_paths: list[str] | None = None,
    forbidden_paths: list[str] | None = None,
    cache_key_refs: list[dict] | None = None,
    prior_pr_refs: list[dict] | None = None,
    qa_evidence_refs: list[str] | None = None,
    known_risks: list[dict] | None = None,
    manual_checks_required: list[str] | None = None,
    context_freshness_status: str = "fresh",
    context_freshness_last_verified: str = "none",
    invalidation_inputs: dict | None = None,
    requested_context_sections: list[str] | None = None,
    output_preferences: dict | None = None,
    created_from_agent: str = "",
    created_from_hook: str = "before_plan",
    created_from_template: str = "",
) -> dict:
    """Build a validated context-pack-inputs dict.

    Parameters are explicit caller-provided inputs.  The function does
    NOT read the filesystem, run Git, or infer values from the environment.

    Returns
    -------
    dict
        A validated dict conforming to schemas/context-pack-inputs.schema.yml.
    """
```

### Helper functions

```python
def normalize_context_pack_inputs(raw: dict) -> dict:
    """Normalize a raw dict to canonical form.

    Applies sorting, empty-value omission, and path validation.
    Does NOT modify the input dict in-place.
    """

def validate_context_pack_inputs(raw: dict) -> None:
    """Validate a context-pack-inputs dict.

    Raises ValueError (or a custom exception if applicable) on
    validation failure.  Pure, no I/O.
    """

def context_pack_inputs_error(
    field: str,
    reason: str,
) -> ValueError:
    """Create a structured validation error."""
```

### Error model

Use a simple `ValueError` with a descriptive message. For example:

```
ValueError(
    f"context_pack_inputs validation failed: "
    f"[{field}] {reason}"
)
```

A custom exception class is not needed for this scope. It can be promoted later if the conductor pipeline needs catchable error types.

### Input validation rules

| Field | Validation |
|---|---|
| `pr_id` | Required, non-empty string |
| `task_goal` | Required, non-empty string |
| `source_contracts` | Optional. Each entry non-empty string. |
| `relevant_anchors` | Optional. Each entry non-empty string. |
| `allowed_paths` | Optional. No absolute paths (must not start with `/`). No shell placeholders. |
| `forbidden_paths` | Optional. No absolute paths. No shell placeholders. |
| `cache_key_refs` | Optional. Each ref must be a dict with at least `"namespace"` and `"artifact_kind"`. |
| `context_freshness_status` | Optional, defaults to `"fresh"`. Must be one of `fresh`, `stale`, `in_progress`, `unknown`. |
| `created_from_agent` | Optional. If non-empty, recorded in `created_from.agent`. |

### Anti-fabrication validation

The generator must validate that:
- No shell placeholders (`$(...)`) appear in any string field.
- No old project names/examples appear (enforced by review, not by runtime validation — runtime cannot know all forbidden patterns).

### Determinism

- Lists are sorted where order is not semantically meaningful: `source_contracts`, `relevant_anchors`, `allowed_paths`, `forbidden_paths`, `prior_pr_refs` (by key), `known_risks` (by `id` key if present), `requested_context_sections`, `qa_evidence_refs`.
- Map keys are sorted lexicographically for `invalidation_inputs`, `output_preferences`, and `cache_key_ref` entries (`cache_key_ref` dicts are not reordered — caller specifies structure).
- Empty lists are omitted from the output dict.
- Empty strings are omitted from the output dict.
- `None` values are omitted from the output dict.

### Serialization

The generator returns plain Python dicts. No YAML/JSON dependency required. The dict can be serialized with `json.dumps(sort_keys=True)` for test assertions or file writing by a future caller.

The generator does NOT write files in this PR. Writing `.project-memory/pr/<pr-id>/context-pack-inputs.yml` is the responsibility of a future Context Steward executor.

### Test coverage

**`build_context_pack_inputs`** (primary function):

- minimal valid input: only `pr_id` and `task_goal` — should produce a valid dict with defaults.
- full input: all optional fields provided — should preserve all values.
- defaults: `context_freshness_status="fresh"`, `context_freshness_last_verified="none"`, empty lists omitted.
- required `pr_id`: empty string raises `ValueError`.
- required `task_goal`: empty string raises `ValueError`.
- absolute path in `allowed_paths`: raises `ValueError`.
- absolute path in `forbidden_paths`: raises `ValueError`.
- shell placeholder in a string field: raises `ValueError`.
- invalid `context_freshness_status`: raises `ValueError`.
- determinism: repeated calls with same inputs produce equal output dicts.
- determinism: lists are sorted in output.
- no I/O: function does not read filesystem.
- no subprocess: function does not call subprocess.
- compatibility with schema: output dict keys match `schemas/context-pack-inputs.schema.yml` field names.

**`normalize_context_pack_inputs`**:

- already-normalized input returns same structure.
- unsorted lists are sorted.
- empty lists are removed.
- empty strings are removed.
- `None` values are removed.
- map keys are sorted.

**`validate_context_pack_inputs`**:

- valid dict passes without exception.
- invalid dict (empty pr_id) raises `ValueError`.
- dict containing old project names raises `ValueError` (optional — see open questions).

## Open Questions

1. **Should `validate_context_pack_inputs` check for old project names at runtime?** **Decision:** No. Old name checking is a review concern, not a runtime validation concern. Runtime cannot know all historical forbidden names. Reviewers and precommit checks handle this. The generator rejects shell placeholders (a universal safety concern) but not historical naming patterns.

2. **Should the generator produce a dataclass or a dict?** **Decision:** Dict. The schema is YAML-based. Returning a dict keeps the function dependency-free (no dataclass import needed) and directly serializable. If a dataclass is needed later for type safety in the conductor pipeline, it can be added as a thin wrapper.

3. **Should `build_context_pack_inputs` write the artifact file?** **Decision:** No. The generator returns a dict. Writing files is the caller's responsibility (Context Steward executor or a future PR). This keeps the generator pure, testable, and dependency-free.

4. **Should the generator be a module of functions or a class?** **Decision:** Module of functions. No state, no lifecycle, no subclassing. Pure functions suit this scope.
